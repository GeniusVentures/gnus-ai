## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| npm registry -> node_modules | Three new `@safe-global/*` packages cross this boundary during install |
| deployer EOA private key -> Safe SDK | Private key material is passed into Safe protocol-kit at runtime |
| local filesystem -> diamonds/<name>/safe-proposals/ | Proposal artifacts (containing calldata but no secrets) are written to disk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08.1-01 | Tampering | npm install of @safe-global/* | mitigate | Task 0 blocking human checkpoint verifies publisher is @safe-global, version is stable, package matches docs.safe.global/sdk-kit before any install |
| T-08.1-02 | Spoofing | proposeSafeTransaction proposer identity | mitigate | Proposer is authenticated via private-key signature on safeTxHash; Safe Transaction Service rejects unsigned/mismatched proposals |
| T-08.1-03 | Information Disclosure | proposerPrivateKey handling | accept | Private key stays in process memory only; never logged, never serialized into the artifact. Artifact contains only safeTxHash, addresses, calldata. |
| T-08.1-04 | Repudiation | Safe proposal submission | mitigate | apiKit.proposeTransaction returns a safeTxHash that the Safe Transaction Service stores with senderAddress — non-repudiable audit trail |
| T-08.1-SC | Tampering | npm/pip/cargo installs | mitigate | Task 0 blocking-human gate; package legitimacy verified on npmjs.com before install. Cannot be auto-approved (workflow.auto_advance ignored). |
</threat_model>
