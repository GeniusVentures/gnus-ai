# Phase 08.1 Plan 02: Safe Proposal Wiring Summary

**SafeProposerRPCDeploymentStrategy intercepts diamondCut via subclass override, proposes to Safe multisig, and writes a local artifact — all gated behind SAFE_PROPOSE=true with fail-fast mainnet guard**

## Performance

- **Duration:** 16m 27s

- **Completed:** 2026-06-30T00:24:22Z
- **Tasks:** 3
- **Files modified:** 3 (1 created, 2 modified)

## Accomplishments

- SafeProposerRPCDeploymentStrategy created as a subclass of RPCDeploymentStrategy, overriding only performDiamondCutTasks to encode diamondCut calldata, propose it to a Safe wallet via the Safe Transaction Service, write a local fallback/audit-trail artifact, and skip canonical deployment-data finalization
- CLI extended with six Safe proposal flags (--safe-propose, --safe-address, --safe-proposer-private-key, --safe-tx-service-url, --safe-api-key, --safe-origin) composed into addCommonOptions so deploy-rpc.ts and upgrade-rpc.ts require zero changes
- RPCDiamondDeployerConfig extended with six safe fields; constructor branches to SafeProposerRPCDeploymentStrategy when safePropose=true; createConfigFromEnv reads all six SAFE\_\* env vars; validateConfig rejects missing/invalid Safe address, missing proposer key, and mainnet direct upgrades without SAFE_PROPOSE
- Sepolia direct upgrades emit a console.warn advisement but are not blocked

## Files Created/Modified

- `scripts/setup/strategies/SafeProposerRPCDeploymentStrategy.ts` - Subclass of RPCDeploymentStrategy; overrides performDiamondCutTasks to propose diamondCut to Safe instead of sending directly; writes local artifact; skips postDiamondCutDeployedDataUpdate and markDeploymentComplete
- `scripts/deploy/rpc/common.ts` - Extended BaseRPCOptions with 6 safe fields; added addSafeOptions() Commander helper composed into addCommonOptions; createRPCConfig passes safe fields through both hardhat and legacy config paths
- `scripts/setup/RPCDiamondDeployer.ts` - Extended RPCDiamondDeployerConfig with 6 safe fields; constructor branches strategy selection; createConfigFromEnv reads safe env vars; validateConfig adds Safe validation and mainnet guard; deployDiamond adds sepolia direct-upgrade warning

## Decisions Made

- IDiamondCut ABI uses a local `kIDiamondCutAbi` constant as fallback when Hardhat artifact resolution (getContractArtifact) fails — this prevents compile-time coupling to Hardhat artifact paths while still preferring the artifact-resolved ABI when available
- Composed addSafeOptions inside addCommonOptions (modifying the Command in-place) rather than requiring call-site changes to deploy-rpc.ts, upgrade-rpc.ts, or any other script that calls addCommonOptions
- Sepolia direct-upgrade warning uses `console.warn` at deploy-time rather than blocking — testnet flexibility preserved while nudging toward Safe proposals
