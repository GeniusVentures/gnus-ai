CRITICAL — actual performDiamondCutTasks implementation (node_modules/@diamondslab/diamonds/dist/strategies/RPCDeploymentStrategy.js lines 505-580):

- Reads `const deployedDiamondData = diamond.getDeployedDiamondData()` and `const diamondAddress = deployedDiamondData.DiamondAddress`.
- Loads IDiamondCut artifact via `getContractArtifact("IDiamondCut", diamond)`.
- Builds `const diamondContract = new ethers.Contract(diamondAddress, diamondCutArtifact.abi, this.signer)`.
- Calls `const [initCalldata, initAddress] = await this.getInitCalldata(diamond)` and `const facetCuts = await this.getFacetCuts(diamond)`.
- Builds `const facetSelectorCutMap = facetCuts.map(fc => ({ facetAddress: fc.facetAddress, action: fc.action, functionSelectors: fc.functionSelectors }))`.
- Sends the tx: `const tx = await diamondContract.diamondCut(facetSelectorCutMap, initAddress, initCalldata, { gasLimit, gasPrice })`.
- Waits: `const receipt = await tx.wait()`.
- Then: `this.updateStepStatus(...)`, `await this.postDiamondCutDeployedDataUpdate(diamond, tx.hash)`, `await this.executeInitializerFunctions(diamond)`, `this.store.markDeploymentComplete()`.

THE OVERRIDE: Reuse everything UP TO building facetSelectorCutMap and getting initCalldata/initAddress. Then instead of `diamondContract.diamondCut(...)`:

- `const calldata = diamondContract.interface.encodeFunctionData('diamondCut', [facetSelectorCutMap, initAddress, initCalldata])`
- `const result = await proposeSafeTransaction({ chainId: BigInt(this.chainId), rpcUrl: this.rpcUrl, safeAddress: this.safeAddress, proposerPrivateKey: this.proposerPrivateKey, safeTxServiceUrl: this.safeTxServiceUrl, safeApiKey: this.safeApiKey, to: diamondAddress, value: '0', data: calldata, operation: OperationType.Call, origin: this.safeOrigin })`
- `await writeSafeProposalArtifact({ diamondName: this.diamondName, networkName: this.networkName, chainId: this.chainId, safeAddress: this.safeAddress, proposerAddress: result.proposerAddress, target: diamondAddress, value: '0', operation: OperationType.Call, safeTxHash: result.safeTxHash, calldata, origin: this.safeOrigin })`
- Log the success block (spec lines 833-845).
- DO NOT call `postDiamondCutDeployedDataUpdate` or `markDeploymentComplete` — the Safe tx has not executed yet. Optionally call `this.updateStepStatus(diamondCutStepName, 'proposed_to_safe', result.safeTxHash, diamondAddress)` if the step store accepts custom statuses; otherwise skip step-status bookkeeping.
