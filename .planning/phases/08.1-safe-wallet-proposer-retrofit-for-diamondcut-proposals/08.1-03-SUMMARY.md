## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed sinon double-wrap error on validateConnection stub**
- **Found during:** Task 1
- **Issue:** Both RPCDeploymentStrategy.prototype.validateConnection and SafeProposerRPCDeploymentStrategy.prototype.validateConnection were stubbed, but the child inherits from the parent — sinon rejected the second stub as "already wrapped"
- **Fix:** Removed the child stub; the parent stub covers both classes via the prototype chain
- **Files modified:** `test/unit/rpc/RPCDiamondDeployerSafePropose.test.ts`
- **Commit:** `7104905`

**2. [Rule 1 - Bug] Fixed non-hex address fixtures causing ethers encodeFunctionData failure**
- **Found during:** Task 2
- **Issue:** DIAMOND_ADDRESS `0xdiamond...` and facet address `0xfacet...` contained non-hex characters, causing ethers v6 address validation to fail during diamondCut calldata encoding
- **Fix:** Replaced with valid all-numeric hex addresses (`0xd1a0...`, `0x0000...1111`)
- **Files modified:** `test/unit/rpc/SafeProposerRPCDeploymentStrategy.test.ts`
- **Commit:** `d54818a`

**3. [Rule 1 - Bug] Fixed init calldata stub containing non-hex string**
- **Found during:** Task 2
- **Issue:** `getInitCalldata` stub returned `'0xinitcalldata'` which ethers rejected as invalid bytes
- **Fix:** Changed to valid hex `'0xdeadbeef'`
- **Files modified:** `test/unit/rpc/SafeProposerRPCDeploymentStrategy.test.ts`
- **Commit:** `d54818a`

**4. [Rule 1 - Bug] Fixed SafeApiKit constructor requiring apiKey when txServiceUrl not set**
- **Found during:** Task 2
- **Issue:** SafeApiKit constructor throws "apiKey is mandatory when txServiceUrl is not defined" even though proposeTransaction is stubbed — the constructor runs before the stub takes effect
- **Fix:** Added `safeTxServiceUrl: 'http://localhost:8000/txs'` to test inputs to bypass the apiKey requirement
- **Files modified:** `test/unit/safe/proposeSafeTransaction.test.ts`
- **Commit:** `d54818a`

**5. [Rule 1 - Bug] Fixed case-sensitive address comparison in proposeSafeTransaction tests**
- **Found during:** Task 2
- **Issue:** `ethers.Wallet.getAddress()` returns checksummed addresses, but PROPOSER_ADDRESS constant was fixed-case; direct comparison failed on case mismatch
- **Fix:** Changed assertions to use `.toLowerCase()` comparison for both values
- **Files modified:** `test/unit/safe/proposeSafeTransaction.test.ts`
- **Commit:** `d54818a`
