  <action>
Add a `supportsInterface()` override to DiamondInitFacet following the GNUSBridge/GNUSNFTFactory pattern. The implementation must check both the parent contract's interface support AND `LibDiamond.diamondStorage().supportedInterfaces`.

Insert after the `onlySuperAdminRole` modifier block is removed (after Task 1) and before `diamondInitialize250()`. Use the `virtual override` pattern since `AccessControlEnumerableUpgradeable` provides `supportsInterface`. Since the post-Task-1 contract inherits from `GeniusAccessControl` → `AccessControlEnumerableUpgradeable`, there is only one parent with `supportsInterface`, so `super` suffices (no diamond ambiguity requiring explicit parent references like `GNUSBridge` has with `ERC1155Upgradeable, AccessControlEnumerableUpgradeable`).

Add:

```solidity
/// @notice Checks if the contract supports a specific interface.
/// @dev Overrides ERC-165 to include Diamond storage supported interfaces.
/// @param interfaceId The interface identifier to check.
/// @return True if the interface is supported, false otherwise.
function supportsInterface(bytes4 interfaceId) public view virtual override returns (bool) {
    return super.supportsInterface(interfaceId) ||
        (LibDiamond.diamondStorage().supportedInterfaces[interfaceId] == true);
}
```

`LibDiamond` is already imported at line 7 (`import "contracts-starter/contracts/libraries/LibDiamond.sol"`). No new imports needed.
</action>
