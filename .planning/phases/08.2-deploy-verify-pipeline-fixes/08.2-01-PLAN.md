# Phase 08.2 Plan 01: Deploy-Verify Pipeline Fixes + BridgeOut Sepolia Upgrade

**Phase Goal:** Fix the Safe upgrade pipeline bugs, deploy the updated GNUSBridge (bridgeOut 5-arg signature) to Sepolia, verify on Etherscan, and harden the tooling for future upgrades.

## Tasks

### T1: Fix `addCommonOptions` — safe-propose flag was dead code
- **File:** `scripts/deploy/rpc/common.ts`
- **Bug:** `addCommonOptions` had a premature `return` before `addSafeOptions(command)`, so `--safe-propose`, `--safe-address`, etc. were never registered as CLI flags
- **Fix:** Remove the early return so the `.option()` chain continues into `addSafeOptions`
- **Status:** ✓ Complete

### T2: Fix chainId validation order for Safe-propose mode
- **File:** `scripts/setup/RPCDiamondDeployer.ts`
- **Bug:** `validateConfig` ran before chainId auto-detection in `getInstance`. When `.env` sets `RPC_URL`, the legacy config path sets `chainId: 0`, and validation fails with "A valid chainId is required when SAFE_PROPOSE=true"
- **Fix:** Move provider setup + chainId auto-detection before `validateConfig` call
- **Status:** ✓ Complete

### T3: Fix `confirmDeployment` selector matching for replaced facets
- **File:** `scripts/safe/confirmDeployment.ts`
- **Bug:** Used exact selector string matching — couldn't match a replaced facet when selectors changed (e.g., GNUSBridge `0xe26d65a6` → `0xa5a5aa20`)
- **Fix:** Switch to majority overlap matching (≥60%) so facets with minor selector changes still match. Also record unrecognised facets under synthetic `Unknown-<addr>` keys instead of silently dropping them
- **Status:** ✓ Complete

### T4: Create `yarn safe:upgrade` orchestrator
- **Files:** `scripts/safe/safeUpgrade.ts`, `package.json`
- **New:** 5-step orchestrator: propose → wait for Safe execution → confirm on-chain → selector validation → Etherscan verification
- **Commands:** `yarn safe:upgrade`, `yarn safe:check`, `yarn safe:confirm`
- **Status:** ✓ Complete
